# OmniOS r151054 / r151056 / r151058 / r151058-build unattended-install opts.
#
# Every step waits for a screen that actually stays on screen long enough
# to OCR. No fall-through-on-timeout hacks: if a screen never appears,
# the build will deadlock here so the real cause is exposed.

Please select a keyboard | enter

#Welcome to the OmniOS installer
Welcome to | enter

Select disks using arrow keys and space bar | space; enter;

# 3 enters land on the multi-field ZFS root-pool config screen, accepting
# the rpool-name / partitioning / compression defaults, then "Continue with
# the installation". The installer advances straight to the timezone region
# picker, which stays on screen for the next step to match.
customise aspects of the ZFS Root Pool | enter; enter; enter

# Do NOT add an "OmniOS | enter" step here. The "OmniOS v11 r151058" banner
# is drawn at the top of EVERY installer screen, so waitForText("OmniOS")
# matches whatever screen is up when the step runs -- a race. In CI it fired
# on the continent picker, consumed the continent selection, and desynced
# every later step (build hung forever at "Is the above information OK?").
# There is no separate hostname prompt in this installer; the hostname is set
# at the very end by the "string h" config-menu step below.

# Timezone region picker ("Please identify a location ... Please select a
# continent or ocean") -- enter accepts the highlighted default and moves
# on to the timezone confirmation screen.
Please select a continent | enter

# Timezone confirmation: "Is the above information OK?" with the TZ summary
# above it. Enter accepts and starts the actual install.
the above information | enter

# Post-install: "Once back at the main menu, ... rpool now has a working
# and mounted boot environment, per above". Stays on screen for the whole
# install duration (several minutes), then waits for enter to dismiss it
# back to the installer main menu.
back at the main menu | enter

# Installer main menu (Welcome to OmniOS installer / Shell / Reboot / Halt).
# The default item on a finished install is Reboot, so enter reboots the
# VM into the freshly installed disk and first-boot init takes over.
Welcome to the OmniOS installer | enter


# --- First-boot OmniOS configuration menu (Network / SSH / etc.) ---
# Key used as "onfigure" so the leading "C" -- prone to OCR drift -- is
# outside the match window.

# Configure Network -> DHCP. Hot key "c" twice (Configure, then DHCP).
onfigure | string c; enter;string c; enter; enter

# Enable SSH, then Halt (hot keys s, s, h). build.py polls VM-down next.
onfigure | string s;string s; enter; enter;  string h; enter; sleep 10
