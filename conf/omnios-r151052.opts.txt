# OmniOS r151052 unattended-install opts.
#
# Every step waits for a screen that actually stays on screen long enough
# to OCR. No fall-through-on-timeout hacks: if a screen never appears,
# the build will deadlock here so the real cause is exposed.

Please select a keyboard | enter

#Welcome to the OmniOS installer
Welcome to |  enter

# Disk-select screen. PaddleOCR splits the heading into "Select" and "disks
# using arrow keys and space bar" as separate lines, so match the part that
# stays on one line. space selects the only disk (c4t0d0), enter proceeds.
disks using arrow keys and space bar | space; enter;

# ZFS root-pool config screen: ">>> Continue <<<" is preselected at the top,
# so ONE enter accepts the recommended rpool-name / partitioning / compression
# defaults and proceeds to the hostname dialog. PaddleOCR reads the title's
# "ZFS" as "ZFs", so match the case-stable prefix "customise aspects of the".
customise aspects of the | enter

# "Enter the system hostname" dialog (text field pre-filled "omnios", <OK>
# preselected). With VM_OCR=paddle, PaddleOCR's neural recognizer reads this
# dim dialog title cleanly, so wait for "system hostname" and press enter to
# accept <OK>. (tesseract saw only the bright "OmniOS" banner here -- and that
# banner is on every screen, so matching it raced; PaddleOCR removes the need
# for any banner / colour-channel trick.)
system hostname | enter

# Timezone region picker ("Please identify a location ... Please select a
# continent or ocean") -- enter accepts the highlighted default and moves
# on to the timezone confirmation screen.
Please select a continent | enter

# Timezone confirmation: "Is the above information OK?" with the TZ summary
# above it. Enter accepts and starts the actual install.
the above information | enter

# Post-install "Installation Complete" screen ("rpool now has a working and
# mounted boot environment ..."). Stays up for the whole install (minutes).
# PaddleOCR fragments the "Once back at the main menu ..." sentence word by
# word, so match the contiguous line "now has a working and mounted boot".
# Enter dismisses it to the installer main menu.
now has a working and mounted boot | enter

# Installer main menu. Its default highlighted item is "Configure the
# installed OmniOS system", so enter opens the first-boot configuration menu
# (Network / SSH / etc.) -- it does NOT reboot. Match "Configure the
# installed", which is unique to this menu and free of the OmniOS/Omnios case
# drift PaddleOCR shows in the banner/title.
Configure the installed | enter


# --- First-boot OmniOS configuration menu (Network / SSH / etc.) ---
# Key used as "onfigure" so the leading "C" -- prone to OCR drift -- is
# outside the match window.

# Configure Network -> DHCP. Hot key "c" twice (Configure, then DHCP).
onfigure | string c; enter;string c; enter; enter

# Enable SSH, then Halt (hot keys s, s, h). build.py polls VM-down next.
onfigure | string s;string s; enter; enter;  string h; enter; sleep 10
